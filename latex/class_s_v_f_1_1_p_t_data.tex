\subsubsection*{template$<$typename Key, typename Datum, typename Data$>$\newline
class SVF\+::\+PTData$<$ Key, Datum, Data $>$}

Basic points-\/to data structure Given a key (variable/condition variable), return its points-\/to data (pts/condition pts) It is designed flexible for different context, heap and path sensitive analysis Context Insensitive Key -\/-\/$>$ Variable, Data -\/-\/$>$ Points\+To Context sensitive\+: Key -\/-\/$>$ \mbox{\hyperlink{class_s_v_f_1_1_cond_var}{Cond\+Var}}, Data -\/-\/$>$ Points\+To Heap sensitive\+: Key -\/-\/$>$ Variable Data -\/-\/$>$ \mbox{\hyperlink{class_s_v_f_1_1_cond_points_to_set}{Cond\+Points\+To\+Set}} Context and heap sensitive\+: Key -\/-\/$>$ \mbox{\hyperlink{class_s_v_f_1_1_cond_var}{Cond\+Var}}, Data -\/-\/$>$ \mbox{\hyperlink{class_s_v_f_1_1_cond_points_to_set}{Cond\+Points\+To\+Set}}

This class is abstract to allow for multiple methods of actually storing points-\/to sets. Key\+: \char`\"{}owning\char`\"{} variable of a points-\/to set. Datum\+: elements in points-\/to sets. Data\+: the points-\/to set; a collection of Datums.
